Documentation
There are currently three types of documentation for SuMPF. This is due to the fact, that it has been tried to generate most of the documentation automatically from the docstrings. And none of the tools has been perfect for everything.
Doxygen The documentation for the classes has been created with Doxygen.
The class index can be found here.
If the documentation is not there yet, it might need to be built with Doxygen. This can be done with the provided Makefile.
ModuleDoc While doxygen is great for documenting classes, I have not got it working for the functions and python submodules of SuMPF. So their docstrings have been extracted by the self written script ModuleDoc.
ModuleDoc only documents accessable components of a module. For example, it does not show abstract base classes that cannot be accessed by importing the module.
The documentation can be found here. If the documentation is not there yet, it might need to be built with ModuleDoc. This can be done with the provided Makefile.

Code examples There are almost no code examples in the documentation, but there are some interesting places in the source code to look for examples.
  • There is an "_examples" folder in the sources for SuMPF, which some examples programs are given. These examples are meant to give code examples, show some features of SuMPF and to provide some potentially useful functionalities.
  • Shorter and more simple examples can be found in the unit tests.
  • There is one example for the connectors in the documentation for the connector decorators. This example might be easier than the unit tests.

What's Monkeyforest? Monkeyforest is a cool measurement system from Four Audio. I had the honour to use it in a lab at the university and was so impressed by the possibilities of acoustical measurements, that I wanted to have such a system as well.
Monkeyforest has two significant drawbacks though:
  • It is not free software, which, in my opinion, disqualifies it for productive work.
  • And it is DOS based. This somewhat limits the possibilities of user interaction. And the support for modern audio hardware might be poor.
That's why I started SuMPF.
Never the less, besides the immense capabilities of Monkeyforest, the DOS look-and-feel only adds to the great style of this software.
