Detailed Description

A class for the convolution of Signals.

It uses either numpy.convolve or a multiplication in the frequency domain.

The input Signals must have the same sampling rate and channel count. If the convolution mode SPECTRUM is used, the signals must also have the same length.

Sets the convolution mode.

The mode can be one of the following:

  • ConvolveSignals.FULL for full length convolution. If M and N are the lengths of the input signals, the output's length will be M+N-1.
  • ConvolveSignals.SAME for an output with the same length as the longer input. (output_length=max(M,N))
  • ConvolveSignals.VALID for an output length of max(M,N)-min(M,N)+1
  • ConvolveSignals.SPECTRUM to do the convolution in the frequency domain. This will be a circular convolution which requires both input signals to have the same length. This will also be the length of the output signal. See help(numpy.convolve) for more details.

    modeone of the modes from the description
