Detailed Description

A decorator that marks a method as an output connector.

These connections can be used to automatically update a processing chain when a value has changed. The decorated method must not take any arguments. This decorator can also enable caching, in which case the return value is only recalculated when the respective setter methods have been called. Please note that caching might lead to wrong return values if not all setter methods notify the output properly. It is also problematic if the getter changes the state of its object.
